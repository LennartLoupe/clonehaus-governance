# Integrating Clonehaus Governance with N8N

N8N is a good fit when you want governance embedded into workflow automation instead of application code. Governance matters here because automation can move quickly from draft assistance to real operational impact, especially when a workflow can message people, update systems, or trigger follow-on actions without a developer in the loop.

## Prerequisites

- Clonehaus Governance Plugin installed
- N8N available in your environment
- Anthropic API access
- Slack credential ready if you want escalation messages
- Agent Charter created with `/clonehaus:create-charter`

## Export Your Governance Code

```text
/clonehaus:export-context --framework=n8n
```

## What Gets Generated

The export generates a workflow JSON file like:

- `clonehaus_sales_follow_up_n8n.json`

The file includes:

- A governance block injected at the start of the workflow
- A function node that validates authority levels before the model call
- An Anthropic API request node using `claude-sonnet-4`
- An escalation decision branch
- Two terminal paths: execute response or escalate to Slack

Approximate size:

- 150-250 lines of JSON depending on formatting and charter detail

## Integration Steps

### Step 1: Import the generated workflow

In N8N:

1. Open the workflow editor.
2. Choose `Import from File`.
3. Select the JSON file generated by `/clonehaus:export-context --framework=n8n`.
4. Confirm the node layout renders correctly.

### Step 2: Configure credentials and environment

The generated workflow expects:

- `ANTHROPIC_API_KEY` available to the HTTP Request node
- A Slack credential for the escalation node

For Anthropic:

- Set `ANTHROPIC_API_KEY` in your N8N environment or update the request node if you use a different secret pattern.

For Slack:

- Open the `Escalate to Human` node
- Attach your Slack credential
- Confirm the target channel, defaulting to `#agent-escalations`

### Step 3: Understand the 5-stage flow

The workflow follows five logical stages:

1. `Input Validation`
2. `Governance Check`
3. `Claude API Call`
4. `Escalation Decision`
5. Terminal outcome:
   `Execute Response` or `Escalate to Human`

The generated JSON uses 6 concrete nodes because the final stage splits into two different outcomes.

### Step 4: Feed request metadata into the workflow

The governance logic expects useful fields on the incoming payload.

Example webhook payload:

```json
{
  "query": "Draft a follow-up email after today's sales call.",
  "decision_level": "ADVISORY",
  "data_classification": "INTERNAL",
  "recipient_scope": "CLIENT_ROUTINE",
  "system_access": "WRITE_LIMITED",
  "risk_flags": [],
  "stakeholders": []
}
```

### Step 5: Test the governed workflow

Start with a permitted request, then force an escalation.

Allowed example:

```json
{
  "query": "Summarize the prospect's objections and create CRM follow-up notes.",
  "decision_level": "ADVISORY",
  "data_classification": "INTERNAL",
  "recipient_scope": "CLIENT_ROUTINE",
  "system_access": "WRITE_LIMITED"
}
```

Escalation example:

```json
{
  "query": "Offer a 20% discount and send the revised commercial terms.",
  "decision_level": "TRANSACTIONAL",
  "data_classification": "INTERNAL",
  "recipient_scope": "CLIENT_ROUTINE",
  "system_access": "WRITE_LIMITED",
  "force_escalation": true
}
```

## Key Features

**Authority Enforcement:**
The `Governance Check` function node compares the requested decision, data, communication, and system levels against the exported charter values before the Anthropic node runs.

**Escalation Handling:**
The `Escalation Decision` node routes flagged work to Slack instead of continuing as if the workflow were safe to execute automatically.

**Customization:**
You can adjust the trigger source, webhook payload shape, Slack channel, and downstream response handling. You should not remove the governance function node or bypass the escalation branch unless you intend to change the charter-backed control path.

## Example Usage

Minimal usage pattern:

1. Import the generated workflow JSON.
2. Trigger it from a webhook or CRM event.
3. Send a payload like this:

```json
{
  "query": "Draft a renewal follow-up based on the account notes.",
  "decision_level": "ADVISORY",
  "data_classification": "INTERNAL",
  "recipient_scope": "CLIENT_ROUTINE",
  "system_access": "WRITE_LIMITED",
  "risk_flags": []
}
```

If the request stays inside the charter, the response path continues through `Execute Response`. If not, the workflow posts to Slack with the escalation reason.

## Troubleshooting

**Common Issue 1: Anthropic request returns 401 or 403**
Check that `ANTHROPIC_API_KEY` is available to N8N and that the `Claude API Call` node is sending the header correctly.

**Common Issue 2: Slack escalation never sends**
Open the `Escalate to Human` node and confirm the Slack credential is attached. Also verify the channel exists and your bot can post there.

**Common Issue 3: Everything escalates immediately**
Inspect the payload arriving at `Governance Check`. A mismatched level like `READONLY` instead of `READ_ONLY` will be treated as unknown and can trigger an error.
