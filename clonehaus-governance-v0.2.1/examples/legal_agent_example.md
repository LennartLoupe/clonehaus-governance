# Legal Agent Example

Complete walkthrough showing how to use the Clonehaus Governance Plugin to create and deploy a governed legal agent.

## Scenario

An in-house legal team wants an agent to review NDAs and commercial contracts before attorneys spend time on them. The team wants faster first-pass analysis, but they also need clear authority boundaries, strong escalation rules, and documentation that supports EU AI Act readiness.

## Step 1: Create the Charter

Command:

```text
/clonehaus:create-charter
```

Complete flow:

```text
Claude: I'll help you create an Agent Charter. We’ll define what this agent can do, what it cannot do, and when it needs a human.

Step 1 of 6: Agent context
What kind of agent is this, and what is it meant to help with?

User: Legal review for commercial contracts

Claude: What decisions or tasks do you expect this agent to handle most often?

User: Review NDAs and standard commercial agreements, flag risky clauses, and prepare an internal recommendation for legal counsel.

Claude: Step 2 of 6: Decision authority
How much decision-making power should this agent have?

1. L1 - Informational
2. L2 - Advisory
3. L3 - Operational
4. L4 - Transactional
5. L5 - Strategic

User: L2 - Advisory

Claude: Step 3 of 6: Data access
What data should this agent be allowed to read, write, or reference?

1. L1 - Public
2. L2 - Internal
3. L3 - Confidential
4. L4 - Restricted

User: L3 - Confidential. It can access NDAs, customer contracts, fallback clauses, and the internal playbook. No privileged board materials.

Claude: Step 4 of 6: External communication
Who can this agent communicate with outside the organization, if anyone?

1. L1 - Internal Only
2. L2 - Client Routine
3. L3 - Advisory
4. L4 - Partner
5. L5 - Public
6. L6 - Leadership

User: L1 - Internal only

Claude: Step 5 of 6: System access
What systems should this agent be able to interact with?

1. L1 - Read-Only
2. L2 - Write (Limited)
3. L3 - Write (Broad)
4. L4 - Admin

User: L1 - Read-only. It can read the contract repository and legal knowledge base, but it cannot edit documents or approve anything.

Claude: Step 6 of 6: Escalation triggers
When should this agent stop and ask a human?

[ ] Scope boundary
[ ] Risk or sensitivity
[ ] Ambiguity
[ ] Key stakeholder
[ ] Authority ceiling
[ ] Custom threshold

User: scope boundary, risk or sensitivity, ambiguity, key stakeholder, liability > $500K
```

## Step 5: Deploy

Drop the generated LangChain file into the service that handles contract review requests, point it at your Anthropic key, and pass action metadata with each request so the governance guard can enforce the charter before the model runs. In practice, this usually sits in front of your document intake flow or internal legal assistant endpoint.

## What This Charter Achieves

- ✓ Gives the legal team a fast first-pass contract review agent without letting it approve or send anything externally.
- ✓ Creates a documented authority boundary that supports EU AI Act work across risk management, transparency, and human oversight.
- ✓ Ensures high-risk contracts escalate quickly when liability, ambiguity, or stakeholder sensitivity shows up.
