# Finance Agent Example

Complete walkthrough showing how to use the Clonehaus Governance Plugin to create and deploy a governed finance agent.

## Scenario

A finance ops team wants an agent to triage incoming invoices, categorize them, and route them to the right approver. The team wants to automate routine work, but they do not want the agent approving payments, changing accounting controls, or operating without clear escalation rules.

## Step 1: Create the Charter

Command:

```text
/clonehaus:create-charter
```

Complete flow:

```text
Claude: I'll help you create an Agent Charter. We’ll define what this agent can do, what it cannot do, and when it needs a human.

Step 1 of 6: Agent context
What kind of agent is this, and what is it meant to help with?

User: Finance ops agent for invoice triage and approval routing

Claude: What decisions or tasks do you expect this agent to handle most often?

User: Categorize invoices, identify missing fields, route to the correct approver, and update invoice status after checks are complete.

Claude: Step 2 of 6: Decision authority
How much decision-making power should this agent have?

User: L3 - Operational

Claude: Step 3 of 6: Data access
What data should this agent be allowed to read, write, or reference?

User: L3 - Confidential. It can access invoices, vendor records, PO references, and internal finance procedures.

Claude: Step 4 of 6: External communication
Who can this agent communicate with outside the organization, if anyone?

User: L1 - Internal only

Claude: Step 5 of 6: System access
What systems should this agent be able to interact with?

User: L2 - Write limited. It can update invoice status and routing fields, but it cannot change bank data or approve payments.

Claude: Step 6 of 6: Escalation triggers
When should this agent stop and ask a human?

User: scope boundary, risk or sensitivity, ambiguity, key stakeholder, authority ceiling
```

## Step 2: Review the Charter

Charter summary output:


## Step 5: Deploy

Use the generated CrewAI agent inside your invoice intake or AP orchestration process, attach the tools that read invoice metadata and update status fields, and keep payment approval outside the agent entirely. This gives finance ops real automation on routine triage without weakening human control over money movement.

## What This Charter Achieves

- ✓ Automates routine invoice triage and routing so finance ops can spend more time on exceptions.
- ✓ Keeps the agent inside a narrow operational lane instead of letting it approve payments or modify sensitive controls.
- ✓ Builds a repeatable workflow that still escalates ambiguity, sensitive cases, and authority-limit situations to a human.
